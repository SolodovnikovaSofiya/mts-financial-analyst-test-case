# Тестовое задание: Анализ объёма операций по исполнителям

**Компания:** МТС, блок финансов  
**Вакансия:** Стажёр-аналитик в команду оптимизации процессов

---

## 📌 Задача

Необходимо было определить код подразделения с наибольшим объёмом операций по каждому исполнителю.

**На входе:** таблица с колонками ID документа, ID исполнителя, Код подразделения.

**На выходе:** итоговая таблица с уникальным ID исполнителя, кодом подразделения-лидера, количеством документов и долей от общего объёма.

---

## 🧠 Подход и алгоритмы

Были предложены и описаны **три алгоритмических подхода**:

1. **Группировка и агрегация (Excel)** — выбран для реализации
2. **Сортировка и выбор первого**
3. **Оконные функции (SQL / Python)**

### Почему выбран Excel?
- Прозрачность и удобство проверки для бизнес-заказчика
- Соответствие требованиям вакансии к владению Excel
- Адекватность объёму данных (3 305 исполнителей)

---

## ⚙️ Реализация

### Основное решение (Excel)
- Сводные таблицы: группировка по исполнителю и подразделению
- Формулы: `МАКСЕСЛИ` для поиска максимума, `СУММЕСЛИ` для расчёта доли
- Обработка особых случаев: при равных максимумах — выбор первого по алфавиту

### Альтернативные реализации
- **Python (Pandas):** `groupby()` + `rank()` + `cumcount()`
- **SQL:** `ROW_NUMBER() OVER (PARTITION BY ... ORDER BY ...)`

- Для каждого определён доминирующий код подразделения
- Рассчитана доля документов в топ-подразделении
- Исходные данные не изменены, все преобразования вынесены на отдельные листы
